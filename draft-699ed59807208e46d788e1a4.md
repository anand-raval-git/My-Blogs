The Nautilus DevOps team needs to set up a new EC2 instance that can be accessed securely from their landing host (`aws-client`). The instance should be of type `t2.micro` and named `xfusion-ec2`. A new SSH key with name `id_rsa` should be created on the `aws-client` host under the`/root/.ssh/` folder, if it doesn't already exist. This key should then be added to the `root` user's authorised keys on the EC2 instance, allowing passwordless SSH access from the `aws-client` host.

1.Create ec2 isntance with name `xfusion-ec2 and t2.micro`

![](https://cdn.hashnode.com/uploads/covers/6683e9700c2137ac86599d6e/f9f4cb66-8f7c-44be-9488-0d199083c12c.png align="center")

2.  Use below command in aws-client
    

```shell
ssh-keygen
cd .ssh
cat id_rsa.pub #copy public key
```

![](https://cdn.hashnode.com/uploads/covers/6683e9700c2137ac86599d6e/89b8c163-6085-41d2-8f87-d52c0e085483.png align="center")

3.  Connect into ec2 instance and run below command
    
4.  ```shell
    sudo su
    cd .ssh
    vim authorized_keys #paste public keyhere
    ```
    

![](https://cdn.hashnode.com/uploads/covers/6683e9700c2137ac86599d6e/82a68307-afe6-4fcc-b718-be8277d6cd15.png align="center")

5.  ```shell
    cd /etc/ssh
    vim sshd_conf
    #give permission which is below in image
    ```
    

![](https://cdn.hashnode.com/uploads/covers/6683e9700c2137ac86599d6e/57869a0a-11b6-4381-ac7d-644f392d0fa2.png align="center")

6.  ```shell
    #from aws-client login now
    ssh root@ec2-ip
    ```
