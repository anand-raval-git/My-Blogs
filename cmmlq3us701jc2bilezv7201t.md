The Nautilus DevOps team is strategizing the migration of a portion of their infrastructure to the AWS cloud. Recognizing the scale of this undertaking, they have opted to approach the migration in incremental steps rather than as a single massive transition. To achieve this, they have segmented large tasks into smaller, more manageable units.

For this task, create an EC2 instance with following requirements:

1.  The name of the instance must be `xfusion-ec2`.
    

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1766312087136/2c8e82b4-99e0-4cf6-9aac-2d2535e25001.png align="center")

2.  You can use the `Amazon Linux` AMI to launch this instance.
    

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1766312100406/da16cc21-63d9-4d5d-ba8a-e7bdba805db7.png align="center")

3.  The Instance type must be `t2.micro`.
    

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1766312112524/339ae5ba-9905-413a-b8a1-665f2ba37d94.png align="center")

4.  Create a new RSA key pair named `xfusion-kp`.
    

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1766312065484/db8aa599-1136-4162-9b36-5ad36aba8efe.png align="center")

5.  Attach the default (available by default) security group.
