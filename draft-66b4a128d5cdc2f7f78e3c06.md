## **• What is Docker ?**

In a KYC (Know Your Customer) company, a Python Flask application is used to compare photos and complete KYC processes, storing customer data in a MySQL database. When a DevOps engineer restarts the MySQL container, customer data can be lost if Docker volumes are not used.

Docker volumes are crucial for persisting data generated by and used by Docker containers. By mounting a volume, the data is stored outside the container's filesystem, ensuring it remains intact even if the container is restarted or removed.

In this scenario, the senior DevOps engineer advised using Docker volume mounts to prevent data loss. By defining a volume and mounting it to the MySQL data directory, the data is stored in the volume and persists across container restarts. This ensures that customer data is not lost, maintaining the reliability of the application.

**Example Code for MySQL Database Mount with Volume Command:**

```bash
docker run -d \
  --name mysql \
  -e MYSQL_ROOT_PASSWORD=rootpassword \
  -e MYSQL_DATABASE=kyc_db \
  -e MYSQL_USER=user \
  -e MYSQL_PASSWORD=password \
  -v db_data:/var/lib/mysql \
  mysql:latest
```

**Explanation:**

1. **Container Name**: The `--name mysql` option names the container `mysql`.
    
2. **Environment Variables**: The `-e` options set environment variables for the MySQL root password, database name, user, and user password.
    
3. **Volume Mounting**: The `-v db_data:/var/lib/mysql` option mounts the `db_data` volume to the MySQL data directory (`/var/lib/mysql`). This setup ensures that the data stored in MySQL is persisted in the `db_data` volume.
    
4. **Image**: The `mysql:latest` specifies the MySQL image to use.
    

By using Docker volumes, the KYC company's customer data is safely stored and persists across container restarts, preventing data loss and ensuring the reliability of the application.

## **•** What is docker network ? how to connect multiple container and explain docker network type

Docker network is a system that allows Docker containers to communicate with each other, as well as with the host system and external networks. Docker provides several network drivers to manage how containers connect to each other and to the outside world.

### How to Connect Multiple Containers

To connect multiple Docker containers, you can use Docker networks. When containers are on the same network, they can communicate with each other using their container names as hostnames.

1. **Create a Network**:
    
    ```sh
    docker network create my_network
    ```
    
2. **Run Containers on the Same Network**:
    
    ```sh
    docker run -d --name container1 --network my_network my_image
    docker run -d --name container2 --network my_network my_image
    ```
    

In this setup, `container1` can communicate with `container2` using the hostname `container2`, and vice versa.

### Docker Network Types

1. **Bridge Network**:
    
    * **Default network type**.
        
    * Containers on the same bridge network can communicate with each other.
        
    * Suitable for applications that run on a single host.
        
2. **Host Network**:
    
    * Removes network isolation between the container and the Docker host.
        
    * The container shares the host's network stack.
        
    * Useful for high-performance networking.
        
3. **Overlay Network**:
    
    * Enables communication between containers on different Docker hosts.
        
    * Suitable for multi-host Docker setups, such as Docker Swarm.
        
    * Requires a key-value store (like Consul, etcd, or Zookeeper) for service discovery.
        
4. **Macvlan Network**:
    
    * Assigns a MAC address to each container, making it appear as a physical device on the network.
        
    * Useful for legacy applications that require direct access to the physical network.
        
5. **None Network**:
    
    * Disables all networking for the container.
        
    * Useful for containers that do not require network access.
        

By understanding and utilizing these Docker network types, you can effectively manage container communication and ensure your applications run smoothly in various environments.

## **•** What is docker compose ?

To move a file to another folder in Ubuntu using the command line interface (CLI), you can use the `mv` command. The syntax is as follows:

```bash
mv /path/to/source/file /path/to/destination/folder/
```

For example, if you have a file named `example.txt` in your current directory and you want to move it to a folder named `destination_folder`, you would use:

```bash
mv example.txt /path/to/destination_folder/
```

If the `destination_folder` is in the current directory, you can simply use:

```bash
mv example.txt destination_folder/
```

This command will move `example.txt` into `destination_folder`. If you want to rename the file while moving it, you can specify the new name in the destination path:

```bash
mv example.txt destination_folder/newname.txt
```

This will move `example.txt` to `destination_folder` and rename it to `newname.txt`.

## **•** What is yaml ?

YAML (YAML Ain't Markup Language) is a human-readable data serialization standard that is commonly used for configuration files and data exchange between languages with different data structures. It is designed to be easy to read and write, making it a popular choice for configuration files in various applications, including Docker Compose, Kubernetes, and Ansible.

### Key Features of YAML:

1. **Human-Readable**: YAML is designed to be easily readable by humans, with a syntax that is simple and intuitive.
    
2. **Data Serialization**: YAML can represent complex data structures, including lists, dictionaries, and scalars, making it suitable for a wide range of applications.
    
3. **Indentation-Based**: YAML uses indentation to represent the structure of the data. This makes the format clean and easy to understand but also requires careful attention to indentation levels.
    
4. **Support for Multiple Data Types**: YAML supports various data types, including strings, numbers, booleans, lists, and dictionaries.
    
5. **Comments**: YAML allows comments, which can be added using the `#` symbol. This is useful for adding explanations or notes within the configuration files.
    

### Example YAML File:

```yaml
version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
  db:
    image: mysql:latest
    environment:
      MYSQL_ROOT_PASSWORD: example
    volumes:
      - db_data:/var/lib/mysql

volumes:
  db_data:
```

In this example, the YAML file is used to define a multi-container Docker application with two services: `web` and `db`. The `web` service uses the Nginx image and serves content from a local `html` directory, while the `db` service uses the MySQL image and stores data in a Docker volume named `db_data`.

YAML's simplicity and readability make it a preferred choice for configuration files and data exchange in many modern software projects.
