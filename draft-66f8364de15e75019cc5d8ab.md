Over 30% of all websites on the internet use WordPress as their content management system (CMS). It is most often used to run blogs, but it can also be used to run e-commerce sites, message boards, and many other popular things. This guide will show you how to set up a WordPress blog site.

## Task-01

* As WordPress requires a MySQL database to store its data ,create an RDS as you did in Day 44
    

To configure this WordPress site, you will create the following resources in AWS:

* An Amazon EC2 instance to install and host the WordPress application.
    
* An Amazon RDS for MySQL database to store your WordPress data.
    
* Setup the server and post your new Wordpress app.
    

### Let’s begin with Task-01

WordPress requires a MySQL database to store its data. We'll start by creating an Amazon RDS instance:

1. **Login to AWS Console** and search for "RDS".
    
2. **Create a new database** using the "Standard create" method and select "MySQL" under Engine options.
    
3. **Select "Free Tier"** under Templates.
    
4. **Configure settings**:
    
    * DB instance identifier: `wordpress-db`
        
    * Master username: `admin`
        
    * Provide a password
        
5. **Configure additional settings** like storage, backups, VPC, and security groups.
    
6. **Review and create** the database. This process may take a few minutes.

Next, we will create an EC2 instance to host our WordPress application.

1. **Launch a new EC2 instance**:

**STEP 03: Install and Configure WordPress**

1. **Update the package repository and install necessary packages**:
    
    ```bash
    sudo yum update -y
    sudo yum install epel-release
    sudo yum install https://rpms.remirepo.net/enterprise/remi-release-7.rpm
    sudo yum install yum-utils
    sudo yum-config-manager --enable remi-php72
    sudo yum install httpd php php-mysqlnd
    
    ```
    
    ![](https://cdn.hashnode.com/res/hashnode/image/upload/v1727544262571/b58ccfe4-fad2-4902-92f3-aff5d517e012.png align="center")
    
2. **Start the Apache web server**:
    
    ```bash
     sudo systemctl start httpd
     sudo systemctl enable httpd
    ```
    
    ![](https://cdn.hashnode.com/res/hashnode/image/upload/v1727544305368/1e02ae90-f5da-4dd4-990f-a6f50507a2d6.png align="center")
    
3. **Download and extract WordPress**:
    
    ```bash
     wget https://wordpress.org/latest.tar.gz
     tar -xzf latest.tar.gz
     sudo cp -r wordpress/* /var/www/html/
     sudo chown -R apache:apache /var/www/html/
     sudo chmod -R 755 /var/www/html/
    ```
    
    ![](https://cdn.hashnode.com/res/hashnode/image/upload/v1727544349606/842843d8-5c79-4335-b923-120b7005d73d.png align="center")
    
4. **Configure WordPress to connect to your RDS database**:
    
    * Rename the wp-config-sample.php file:
        
        ```bash
         sudo mv /var/www/html/wp-config-sample.php /var/www/html/wp-config.php
        ```
        
    * Edit the wp-config.php file:
        
        ```bash
         sudo vim /var/www/html/wp-config.php
        ```
        
        Update the database settings:
        
        ```bash
          define('DB_NAME', 'your-database-name');
          define('DB_USER', 'your-database-username');
          define('DB_PASSWORD', 'your-database-password');
          define('DB_HOST', 'your-rds-endpoint');
        ```
        
5. **Install and configure the MySQL client**:
    
    ```bash
     sudo apt install mysql-client -y
    ```
    
    * Connect to your MySQL database:
        
        ```bash
          sudo mysql -h your-rds-endpoint -P 3306 -u your-database-username -p your-database-password
        ```
        
    * Create a database user for WordPress:
        
        ```bash
          CREATE DATABASE wordpress;
          CREATE USER 'wordpress' IDENTIFIED BY 'wordpress-pass';
          GRANT ALL PRIVILEGES ON wordpress.* TO wordpress;
          FLUSH PRIVILEGES;
          EXIT;
        ```
        
6. **Install Apache, PHP, and necessary modules**:
    
    ```bash
     sudo apt-get install apache2 php libapache2-mod-php php-mysql -y
    ```
    
7. **Start and enable the Apache web server**:
    
    ```bash
     sudo systemctl restart apache2
     sudo systemctl enable apache2
    ```
    
8. **Download and configure WordPress**:
    
    **Copy**
    
    **Copy**
    
    ```bash
     sudo wget https://wordpress.org/latest.tar.gz
     sudo tar -xzf latest.tar.gz
     sudo cp -r wordpress/* /var/www/html/
    ```
    
9. **Update the WordPress configuration**:
    
    * Rename and edit the wp-config.php file:
        
        **Copy**
        
        **Copy**
        
        ```bash
          sudo mv /var/www/html/wp-config-sample.php /var/www/html/wp-config.php
          sudo nano /var/www/html/wp-config.php
        ```
        
        Update the database settings:
        
        **Copy**
        
        **Copy**
        
        ```bash
          define('DB_NAME', 'wordpress');
          define('DB_USER', 'wordpress');
          define('DB_PASSWORD', 'wordpress-pass');
          define('DB_HOST', 'your-rds-endpoint');
        ```
        
10. **Set up Authentication Unique Keys and Salts**:
    
    **Copy**
    
    **Copy**
    
    ```bash
    define('AUTH_KEY',         'unique-phrase');
    define('SECURE_AUTH_KEY',  'unique-phrase');
    define('LOGGED_IN_KEY',    'unique-phrase');
    define('NONCE_KEY',        'unique-phrase');
    define('AUTH_SALT',        'unique-phrase');
    define('SECURE_AUTH_SALT', 'unique-phrase');
    define('LOGGED_IN_SALT',   'unique-phrase');
    define('NONCE_SALT',       'unique-phrase');
    ```
    
11. **Restart Apache and complete WordPress installation**:
    
    **Copy**
    
    **Copy**
    
    ```bash
    sudo systemctl restart apache2
    sudo systemctl status apache2
    ```
    
    * Open your web browser and navigate to your EC2 instance's public DNS ([**your-ec2-instance-public-dns**](http://your-ec2-instance-public-dns/)).
        
    * Follow the on-screen instructions to set up WordPress. Enter your site title, username, password, and email address.
        
12. **Log in to your new WordPress site**:
    
    * After completing the setup, log in to your WordPress admin dashboard by navigating to [**your-ec2-instance-public-dns/wp-admin**](http://your-ec2-instance-public-dns/wp-admin).
